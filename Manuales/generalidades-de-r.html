R para Biólogos

Capítulo 2 Generalidades de R

R es un lenguaje de programación, cuyo código de programación es escrito en un texto (archivo) el cual es ejecutado directamente por el intérprete de R, sin necesidad de ser compilado como otros lenguajes (verbigracia, C, C++, Pascal, Fortram, etc.). Al ejecutar un programa, R almacena la(s) estructura(s) de datos, variables, funciones, entre otros en forma de de objetos dentro de la memoria. Así, los resultados de las funciones y procedimientos son almacenados también como objetos los cuales pueden ser examinados, evaluados, almacenados en un archivo para la lectura o impresos.

En resumen R es un lenguaje de programación y un ambiente (software) utilizado para el análisis de datos (información) con una salida gráfica de alta calidad.

RStudio es un ambiente de desarrollo integrado (integrated development environment, IDE) en el cual se puede ejecutar el código de R.

2.0.1 Otros editores–ambientes posibles de utilizar con R

Otra de las posibilidades es el instalar Atom el cual se encuentra disponible para los tres mayores sistemas operativos comerciales actualmente. Atom es un editor de textos que soporta una gran cantidad de diferentes lenguajes, entre ellos R.

Para los usuarios de Linux que prefieren trabajar en un ambiente con menos distracciones se recomienda el uso de emacs y ESS Emacs Speaks Statistics, donde se puede trabajar con el script y la consola de R, generando la salida gráfica en una ventana diferente.

Después de haber instalado R y Rstudio, iniciaremos llamando a RStudio.

2.2 Como usuario de primera ocasión de R

Es necesario saber que R utiliza la memoria y el espacio en disco de la computadora, iniciando, la primera ocasión, en la raíz del disco. Es importante organizar la información en un espacio seleccionado, exclusivamente, para esto la forma más sencilla es el utilizar los subdirectorios.

Pasos:

  • Crear un subdirectorio para los datos y script (No utilizar el subdirectorio R donde se encuentra la instalación del lenguaje)

  • Ponerle un nombre fácil de reconocer

  • Para cada proyecto o grupo de información se pueden crear subdirectorios dentro de éste mismo directorio.

  • En la sesión siguiente ya no tendrá que realizar los pasos anteriores, solo acceder al subdirectorio e iniciar su trabajo.

2.2.1 Script

El script es un archivo donde se encuentra las diferentes funciones que serán utilizadas.

Pasos:

  • File
  • New File
  • R Script (Ctrl + Shift + N)

2.3 Iniciar con el subdirectorio de trabajo

Es el momento de redireccionar Rstudio para que utilice el subdirectorio recién creado, subdirectorio de trabajo

Pasos:

  • Files (ventana inferior derecha)

  • Buscar el subdirectorio recién creado (Ud. le ha puesto el nombre)

  • El directorio se encuentra en blanco (esta vacío)

  • Ir a sección More

  • sobre iluminar Set as a working directory

  • Apretar el botón izquierdo del mouse (mice) o presionar Enter

Ya tiene en éste momento la función para direccionar el subdirectorio de trabajo. Esto se tiene que hacer al inicio de cada sesión

2.3.1 Recordando el working directory (setwd())

Si se trabaja con el mismo script durante varias sesiones, como es el caso de éste curso, se puede insertar la función dentro del script para ser utilizada nuevamente en la siguiente sesión.

Pasos:

  • History (ventana superior derecha)

  • sobre-iluminar la función que inicia con setwd()

  • hacer click en To Source

  • ya se tiene en este momento la función en el script

  • puede salvar (save) su primer script, note que de llama Untiled1 y se encuentra en color rojo

  • haga click en el ícono del disco (ícono azul)

  • siga las instrucciones para salvar su archivo

ya puede utilizar la función para asignar el subdirectorio de trabajo cada ocasión que inicie el trabajo con ese script.

2.3.2 Ejemplo del subdirectorio de trabajo (working directory)

En la asignación del subdirectorio de trabajo se utiliza la función setwd() (ejemplo personal):

setwd("~/Dropbox/mydata/Curso_R_2019")

Se tiene que declarar el subdirectorio de trabajo, esto es importante, ya que es lugar donde se encuentra la(s) base(s) de datos que se quiere analizar, además, donde se quiere que se salve el script y las gráficas que se han realizado durante la(s) sesión(es) de trabajo.

2.3.3 Conocer el subdirectorio de trabajo

Para saber donde se encuentra direccionado R es necesario utilizar la función:

getwd()

En mi caso será:

getwd()
## [1] "C:/Users/Usuario/Dropbox/R/Clases R/libro_intro_R/bookdown-demo-master/bookdown-demo-master"

Puede escribir en la consola la función o escribirla en el script, para que se ejecute haga click en Run o presione las teclas Control+Enter
